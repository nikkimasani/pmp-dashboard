EVM Formulas
Cost Variance (CV)
CV = EV − AC
Positive = under budget. Negative = over budget.
Schedule Variance (SV)
SV = EV − PV
Positive = ahead. Negative = behind schedule.
Cost Performance Index (CPI)
CPI = EV ÷ AC
CPI > 1 = under budget. CPI < 1 = over budget.
Schedule Performance Index (SPI)
SPI = EV ÷ PV
SPI > 1 = ahead. SPI < 1 = behind schedule.
Estimate at Completion (EAC) — 3 Versions
EAC = BAC ÷ CPI (most common)EAC = AC + (BAC − EV)EAC = AC + [(BAC−EV) ÷ CPI]
Know all 3 versions — each is used in different scenarios.
ETC, VAC, TCPI
ETC = EAC − ACVAC = BAC − EACTCPI = (BAC−EV) ÷ (BAC−AC)
Other Key Formulas
PERT (3-Point Estimate)
PERT = (O + 4M + P) ÷ 6Std Dev = (P − O) ÷ 6
O=Optimistic, M=Most Likely, P=Pessimistic
Communication Channels
Channels = n(n−1) ÷ 2
n = number of stakeholders. 10 people = 45 channels.
Float / Slack
Float = LS − ES OR LF − EF
Critical path = zero float. Any delay → project delay.
Expected Monetary Value (EMV)
EMV = Probability × Impact
Used in decision trees. Positive = opportunity, Negative = threat.

📋 Session Notes — Key Study Topics
🔥 High-Yield Topics (Exam Favorites)
Stakeholder Conflict
PMI answer: always meet face-to-face, engage directly, never escalate until you've tried. "Confront" = collaborative problem-solving, not arguing. Smoothing buys time but doesn't resolve. Forcing leaves resentment.
Change Control
Every change goes through the CCB — even "small" changes, even regulatory ones. No work before approval. The PM identifies impact first, THEN submits to CCB. The PM does NOT approve changes themselves.
Agile Team Issues
Scrum boundaries: SM removes impediments. PO owns backlog priority only. Dev team owns HOW. Nobody assigns tasks to the dev team — they self-organize. PO cannot change sprint scope mid-sprint unilaterally.
Ethics Scenarios
Always: confirm facts first, document your findings, report through proper channels. Recuse yourself from any situation with a conflict of interest. Never take gifts that create obligation. Report violations — silence is complicity.
EVM Traps
Most common errors: confusing SV (EV−PV) with CV (EV−AC). Choosing wrong EAC formula. Rule: "current performance continues" → BAC÷CPI. "Re-estimate from here" → AC+(BAC−EV). Know all 3 cold.
Risk Process Order
Sequence: Identify → Qualitative Analysis → Quantitative Analysis → Plan Responses → Implement → Monitor. Cannot respond before analyzing. Residual risk = what remains after response. Secondary risk = new risk CAUSED by your response.
Leadership Style
Tuckman + Situational: Forming=directing, Storming=coaching, Norming=supporting, Performing=delegating. PMI ALWAYS prefers collaborative resolution. Forcing only justified when time is truly critical and all else failed.
Scope vs. Quality
Gold plating = adding features beyond scope (wrong even if well-intended). Grade vs. Quality: low grade by design is OK; low quality is never acceptable. Fitness for use = meets customer's actual need.

🌐 Host on GitHub Pages — Free Public URL
One-time setup, ~5 minutes. Your dashboard becomes accessible at https://yourusername.github.io/pmp-dashboard from any device.
Step 1 — Create GitHub Account
Go to github.com and create a free account if you don't have one.
Step 2 — Create a New Repository
Click + → New repository. Name it pmp-dashboard. Set it to Public. Click Create repository.
Step 3 — Upload the File
In your new repo, click uploading an existing file. Drag PMP_Deploy/index.html into the upload area. Scroll down and click Commit changes.
Step 4 — Enable GitHub Pages
Go to Settings → Pages. Under Source, select Deploy from a branch → Branch: main → folder: / (root). Click Save. Wait ~2 minutes.
Step 5 — Your Live URL
Your dashboard is now live at https://YOUR-USERNAME.github.io/pmp-dashboard. Bookmark it. Access from any device, phone, tablet, or another computer.
⚠ Cross-Device Notes Sync
Notes are stored in localStorage — device-specific. To sync notes between devices:
1. Click Export Notes on device A → saves a .json file.
2. Open the dashboard on device B → click Import Notes → select the file.
Your notes are now on both devices.
💡 Keeping the Site Updated
When improvements are made to the dashboard, re-upload index.html to your GitHub repo (same steps — it will overwrite the old file). GitHub Pages auto-updates within minutes.
